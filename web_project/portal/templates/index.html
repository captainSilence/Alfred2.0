Network Automation by Alfred 2.0


Alfred 2.0 was designed to support DIA requests which typically involve Ciena EDS Switch and Juniper CAR Routers. Current version only supports directly connected EDS swiches and CAR routers. Phase 2 will include additional features at a later time.

Today, intent-based networking capabilities are helping companies keep pace with the demands of digital and cloud. But effectively using those capabilities requires a new mindset and approach to the network focused on automation.



Seeing the network as a companywide software platform means that the network is no longer a set of physical boxes (that is dead), but instead a set of virtual software that can be quickly and easily deployed and distributed (long live the network!).
